# PC 截图工具规划文档

**项目名称**: SnapShot Pro（轻量截图工具）
**文档版本**: v1.0
**创建日期**: 2026-06-14
**目标平台**: Windows 7 / 8 / 8.1 / 10 / 11（32位 & 64位）

---

## 一、项目概述

开发一款轻量级桌面截图工具，支持区域截图、窗口截图，并提供箭头、标注框、文字等基础标注功能，最终打包为单一 `.exe` 文件，无需安装、无需运行时依赖。

### 核心设计原则

- **轻量优先**：运行时内存占用控制在 30MB 以内（待机时低于 10MB）
- **零依赖**：单文件 exe，无需安装 .NET / Python / VC++ 运行时
- **兼容优先**：兼容 Windows 7 SP1 及以上所有主流版本
- **简洁 UI**：操作流程不超过 3 步完成截图+标注+保存

## 二、技术选型

### 推荐方案：C# + WinForms + .NET Framework 4.0

| 维度 | 选择 | 理由 |
|------|------|------|
| 语言 | C# 7.3 | 开发效率高，Win API 调用方便 |
| UI 框架 | WinForms | 轻量，GDI+ 原生支持绘图 |
| 运行时 | .NET Framework 4.0 | Win7 SP1 预装，Win8/10/11 内置 |
| 打包工具 | ILRepack / Costura.Fody | 合并依赖为单一 exe |
| 图像处理 | GDI+ (System.Drawing) | 框架内置，无需第三方库 |
| 快捷键 | Win32 RegisterHotKey | 全局热键注册 |
| 系统托盘 | NotifyIcon | WinForms 内置 |

### 备选方案对比

| 方案 | 内存占用 | Win7 兼容 | 开发难度 | 单 exe |
|------|---------|-----------|---------|--------|
| **C# WinForms .NET4** | ~15MB | ✅ 预装 | 中 | ✅ |
| Python + Tkinter | ~60MB | ⚠️ 需 Python 3.8 | 低 | ✅(较大) |
| C++ Win32 | ~5MB | ✅ | 高 | ✅ |
| Electron | ~150MB | ⚠️ | 低 | ✅(很大) |

**结论**：选用 C# WinForms + .NET Framework 4.0，兼顾开发效率与运行时轻量。

## 三、功能规划

### 3.1 截图功能

#### 区域截图
- 全屏遮罩覆盖后用鼠标拖拽选择矩形区域
- 拖拽时实时显示选区尺寸（像素）
- 支持调整已选区域的边/角控制点
- 支持取消（Esc 键）

#### 窗口截图
- 鼠标悬停时自动高亮检测到的窗口
- 点击即截取对应窗口（包含阴影可选去除）
- 支持最小化窗口的截图（调用 PrintWindow API）

#### 全屏截图
- 一键截取所有显示器（多显示器支持）
- 可选仅截当前主显示器

#### 延时截图
- 支持 2s / 5s / 10s 倒计时后自动截图

### 3.2 标注功能

| 工具 | 功能描述 |
|------|---------|
| 箭头 | 拖拽绘制单向/双向箭头，可调整颜色、粗细 |
| 矩形框 | 拖拽绘制矩形标注框，支持实心/空心、圆角选项 |
| 文字输入 | 点击定位，输入文字，支持字体大小/颜色 |
| 马赛克 | 拖拽覆盖区域打马赛克（隐私保护） |
| 撤销/重做 | Ctrl+Z / Ctrl+Y，最多 20 步历史 |

### 3.3 输出功能

- 复制到剪贴板（Ctrl+C）
- 保存为文件：PNG（默认）/ JPG / BMP
- 自动命名：`Screenshot_yyyyMMdd_HHmmss.png`
- 自定义保存路径（记住上次路径）

### 3.4 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Alt + A` | 区域截图（全局） |
| `Alt + W` | 窗口截图（全局） |
| `Alt + F` | 全屏截图（全局） |
| `Esc` | 取消当前操作 |
| `Enter` | 确认并进入标注 |
| `Ctrl+S` | 保存截图 |
| `Ctrl+C` | 复制到剪贴板 |

## 五、关键模块设计

### 5.1 区域选择遮罩窗口 (SelectionOverlay)

```
策略：全屏透明窗口 + GDI+ 实时绘制
- TopMost = true，覆盖所有窗口
- 背景：半透明黑色遮罩（alpha = 120）
- 选区内：挖空显示原始屏幕内容（XOR 绘制）
- 鼠标样式：十字光标
- 性能：仅在 MouseMove 时触发 Invalidate()，避免全屏重绘
```

### 5.2 截图引擎 (ScreenCapture)

```csharp
// 核心 API 调用链
BitBlt(hdcDest, x, y, w, h, hdcSrc, srcX, srcY, SRCCOPY)
// 多显示器：遍历 Screen.AllScreens，使用虚拟屏幕坐标
// 窗口截图：PrintWindow() 支持最小化窗口
```

### 5.3 标注画布 (AnnotationCanvas)

```
采用「原图层 + 标注层」双缓冲设计：
- originalBitmap：保存原始截图，不可修改
- annotationBitmap：标注叠加层，支持透明
- 显示时合并两层，避免频繁重绘 originalBitmap
- 每次标注操作后将当前 annotationBitmap 推入历史栈
```

### 5.4 内存优化策略

| 策略 | 说明 |
|------|------|
| 及时 Dispose | 截图 Bitmap 使用完立即 Dispose |
| 历史栈限制 | 最多保留 20 步，超出时 Dispose 最旧的 |
| 遮罩窗口复用 | 全局单例，隐藏而非销毁重建 |
| 托盘模式 | 非截图状态仅保留托盘图标，主窗口 Hide |

## 八、打包方案

### 单 exe 打包：Costura.Fody

```xml
<!-- packages.config -->
<package id="Fody" version="6.6.4" />
<package id="Costura.Fody" version="5.7.0" />
```

```xml
<!-- FodyWeavers.xml -->
<Weavers>
  <Costura />
</Weavers>
```

编译命令：
```
msbuild SnapShotPro.sln /p:Configuration=Release /p:Platform=AnyCPU
```

输出：`bin/Release/SnapShotPro.exe`（预计体积 1-3 MB）

### 编译器优化选项
```xml
<Optimize>true</Optimize>
<DebugType>none</DebugType>
<DebugSymbols>false</DebugSymbols>
```

## 十一、风险与注意事项

| 风险 | 说明 | 应对措施 |
|------|------|---------|
| Win7 DPI 支持 | Win7 高 DPI 下坐标偏移 | 清单文件声明 DPI-aware |
| 热键冲突 | 全局热键被其他软件占用 | 注册失败时提示用户重新配置 |
| UAC 权限 | 截取管理员权限窗口需提权 | 可选"以管理员身份运行"模式 |
| 多显示器坐标 | 非主显示器坐标系偏移 | 使用虚拟屏幕坐标统一处理 |
| 内存泄漏 | Bitmap 未 Dispose | 严格 using 语句包裹所有 GDI 资源 |
